# Отчёт о качестве данных

## 1. Введение

**Датасет:** blink.csv
**Источник:** (https://archive.ics.uci.edu/dataset/754/dataset+based+on+uwb+for+clinical+establishments)]  
**Размер:** 609 x 9

**Предметная область:** Данный датасет содержит результаты регистрации сигналов от системы позиционирования, в которой взаимодействуют якоря (anchor) и метки (tag). Каждая строка отражает событие приёма сигнала с указанием идентификаторов устройств, служебных признаков и временных меток.

**Условный заказчик:** Условным заказчиком анализа может выступать команда разработчиков или инженеров, занимающихся отладкой и анализом работы системы позиционирования в реальном времени.

**Возможные задачи ИАД:**
1. Выявление аномалий в работе оборудования (аномальные временные метки, пропуски сигналов);
2. Анализ закономерностей взаимодействия между якорями и метками;
3. Сегментация событий по устройствам и временным интервалам;

## 2. Паспорт датасета

| Признак        | Тип      | Описание              |
| -------------- | -------- | --------------------- |
| id             | int      | Номер записи          |
| anchorID       | category | Идентификатор якоря   |
| tagID          | category | Идентификатор метки   |
| sequenceID     | int      | Номер сигнала         |
| pan            | int      | Служебный параметр    |
| processed_flag | int      | Флаг обработки        |
| timestampToA   | int      | Сырая временная метка |
| Timestamp ToA  | int      | Время в мс            |
| datetime       | datetime | Время события         |

## 3. Основные проблемы качества данных

| № | Тип проблемы                           | Описание         | Потенциальный риск      | Предлагаемое решение    |
|---|----------------------------------------|------------------|-------------------------|-------------------------|
| 1 | Первая строка — данные, а не заголовки | все столбцы      | Нарушает структуру      | Переназначить заголовки |
| 2 | Неверные типы данных                   | числовые столбцы | Анализ невозможен       | Приведение типов        |
| 3 | Дублирующиеся временные столбцы        | timestampToA     | Избыточность            | Оставить один           |
| 4 | Возможные пропуски после конвертации   | числовые         | Потеря данных           | Очистка                 |
| 5 | Категориальные признаки в object       | anchorID, tagID  | Неэффективная обработка | Привести к category     |
